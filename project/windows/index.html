&nbps;How to build a ca2 Framework solution at Windows...

 Check the prerequisites at Windows,
if you haven't done so already.

 Create repository named SolutionProjectsUsingCa2 (replace solution by desired SolutionProjectsUsingCa2 for the solution1) and set it current:

The given example here shows example for 3 (three) online repositories previously created at GitHub:

    https://github.com/yourrepository/SolutionProjectsUsingCa2-windows-solution https://github.com/yourrepository/SolutionProjectsUsingCa2-source https://github.com/yourrepository/SolutionProjectsUsingCa2

mkdir C:\project

C:

cd \project

git clone https://github.com/yourrepository/SolutionProjectsUsingCa2-windows-solution .

 For windows development (with command prompt folder C:\project):

git submodule add https://github.com/ca2/windows-archive archive --recurse-submodules

 Create a place for your SolutionProjectsUsingCa2 source and helper platform neutral (or mostly neutral) projects...

At folder C:\project\source

git submodule add https://github.com/yourrepository/SolutionProjectsUsingCa2-source source

cd C:\project\source

 Your solution "SolutionProjectsUsingCa2" and ca2 Framework common source will be here (with command prompt folder C:\project\source):

At folder C:\project\source

git submodule add https://github.com/ca2/app app

git submodule add https://github.com/ca2/include include

(Optionally for example projects) git submodule add https://github.com/ca2/app-simple app-simple

git submodule add https://github.com/ca2/third third

git submodule add https://github.com/yourrepository/SolutionProjectsUsingCa2 SolutionProjectsUsingCa2

 Your project for App1 (replace App1 by the name of your app project at SolutionProjectsUsingCa2) files would lie at a folder structure like this:

https://github.com/yourrepository/SolutionProjects/_matter/App1/_std/_std (for language neutral or default resources)

https://github.com/yourrepository/SolutionProjects/_matter/App1/en-us/en-us (for language english resources)

...

https://github.com/yourrepository/SolutionProjects/App1 (for source and project files for App1 project)

The Visual Studio solution file is recommended to be put in path below:

C:\project\source\project\_seed\App1.sln

Remote path: https://github.com/yourrepository/SolutionProjects/_seed/App1.sln
